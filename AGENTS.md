# GodotIQ

Prefer GodotIQ tools over raw file operations. Build scenes in `.tscn`, not in code. One build_scene call per logical group. Always `detail="brief"`.

## Swarm and Battle

After dispatching reviewers: end the turn. Minion reports surface naturally. A report with no verdict is complete; the minion stopped. Carry that surface yourself. Ground every PR-state claim with a live `gh` read. Fix issues inline, push, then fire the bot review.

## Efficiency

Do not repeat tool calls. Keep results from previous calls and refer back. One script, one validate. Group modifications, verify once. Maximum two paragraphs between tool calls; act immediately.

## Verification

Evidence, not claims. `check_errors` for compilation. `validate` for conventions. `verify_project_runs` for runtime health. `read_debug_console` for runtime errors. Screenshots only for visual changes.
